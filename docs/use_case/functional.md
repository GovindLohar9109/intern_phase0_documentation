# Functional Use Case

## Overview
The **E-Learning Management System (ELMS)** allows users to learn through free documentation . The system provides role-based access where  users can explore and enroll in courses, while admins can manage all course-related operations.

---

## 1. User Authentication

### 1.1 Login / Register
- New users can **register** or **log in** using their credentials.
- Upon successful login, the system identifies the user's role:
  - **User**
  - **Admin**
- Passwords are securely hashed and stored in the database.

---

## 2. User Role:  User

### 2.1 View All Courses
- After logging in, a  user can view all available courses listed on the website.
- Each course card displays:
  - Course title  
  - Description  
  - view free documentation or add the course to “My Courses”.

### 2.2 Access Free Documentation
- Users can click on the **“Free Documentation”** button to read course material for free.

### 2.3 Add to My Courses
- Users can add interesting courses to **My Courses**.
- Once added, courses appear in the **My Courses** section for future access.

### 2.4 Search to Courses
- User Can search specific Course on the basis of Course Name(Index)
